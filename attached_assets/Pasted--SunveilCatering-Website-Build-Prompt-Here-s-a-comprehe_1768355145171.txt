# SunveilCatering Website Build Prompt

Here's a comprehensive prompt you can feed to your AI agent to build this project:

---

## Project Overview

Build a modern e-commerce website for **SunveilCatering**, a Somali-owned artisan bakery based in Atlanta that sells pastries both direct-to-consumer (by the dozen with delivery) and through local retail partners. The design should blend Blue Apron's clean UX patterns with brand cult psychology principles to create an emotionally compelling experience.

---

### Color System
```

Accent Colors:
- Amber: #ba8919 (heritage storytelling, warmth, highlights)

- Off-white/cream backgrounds
```

### Brand Voice
- Warm, authentic, culturally proud
- Storytelling-forward (Somali heritage is central)
- Community-focused (both local shops and direct customers)
- Premium but accessible

## Technical Stack

- **Hosting**: Azure (Static Web Apps for frontend, Container Apps for API)
- **Database**: Azure SQL Database
- **Caching**: Azure Cache for Redis (cart management, sessions)
- **Payments**: Stripe (Payment Intents API with Stripe Elements)
- **Maps**: Google Maps API for store locator
- **Frontend**: Next.js with TypeScript, Tailwind CSS
- **Image Storage**: Azure Blob Storage with CDN

## Database Schema Requirements

```sql
-- Core tables needed:

products
- id, sku, somali_name, english_name, slug, description, origin_story
- base_price_dozen, base_price_single, category_id
- is_halal, is_vegetarian, is_nut_free
- image_urls (JSON array), is_active, created_at, updated_at

categories
- id, name, slug, description, display_order

partner_locations
- id, name, address, city, state, zip, lat, lng
- phone, hours (JSON), partner_type, is_active
- products_available (JSON array of product IDs)

orders
- id, order_number, customer_id, status
- delivery_address (JSON), delivery_date, delivery_instructions
- subtotal, tax, shipping, total
- stripe_payment_intent_id, created_at, updated_at

order_items
- id, order_id, product_id, quantity, unit_price, line_total

customers
- id, email, password_hash, first_name, last_name
- phone, default_address (JSON), stripe_customer_id
- created_at

cart_sessions (Redis structure, not SQL)
- session_id → { items: [...], created_at, updated_at }
```

## API Endpoints Required

```
Products:
GET    /api/products              - List all products (with filters)
GET    /api/products/:slug        - Single product detail

Categories:
GET    /api/categories            - List categories

Locations:
GET    /api/locations             - List partner locations (with filters)
GET    /api/locations/nearby      - Get locations near lat/lng

Cart:
GET    /api/cart                  - Get current cart
POST   /api/cart/items            - Add item to cart
PATCH  /api/cart/items/:id        - Update item quantity
DELETE /api/cart/items/:id        - Remove item

Checkout:
POST   /api/checkout/delivery-check   - Check ZIP code delivery options
POST   /api/checkout/create-payment   - Create Stripe payment intent
POST   /api/checkout/confirm          - Confirm and create order

Orders:
GET    /api/orders                - List user's orders
GET    /api/orders/:id            - Single order detail

Webhooks:
POST   /api/webhooks/stripe       - Stripe webhook handler

Auth:
POST   /api/auth/register
POST   /api/auth/login
POST   /api/auth/logout
GET    /api/auth/me
```

## Key UX Requirements

1. **Mobile-first responsive design** - 70%+ traffic will be mobile
2. **Lazy loading** for all product images
3. **Skeleton loaders** while content fetches
4. **Persistent cart state** across sessions (Redis + localStorage fallback)
5. **Accessibility**: WCAG 2.1 AA compliance, test green/white contrast ratios
6. **Page transitions**: Smooth, subtle animations
7. **Form validation**: Inline, real-time feedback
8. **Error states**: Friendly error messages with recovery actions

---

## Brand Cult Psychology Elements to Implement

1. **Scarcity**: "Fresh batch slots: X left for tomorrow" on cart/checkout
2. **Exclusivity**: "Small-batch daily" messaging throughout
3. **Origin myth**: Somali heritage story prominently on homepage and product pages
4. **Ritual creation**: "Start your morning with tradition" / "Sunday pastry tradition"
5. **Insider language**: Somali names with pronunciation guides
6. **Community**: Highlight both local partners and direct customers as part of "movement"
7. **Progress gamification**: Dozen box builder with visual progress (5/12 items)
